bValidator documentation & demo

What is bValidator

bValidator is a jQuery plugin for client side form validation. It supports fully customized error messages defined with template and css, many validation methods like date, email, URL, IP address, validation of dinamic forms, customized validation functions and more.

Setup

For basic setup you will need to include into your web page wich contains form you want to validate jquery.bvalidator.js file and, of course, jQuery:

include javascript:

<script type="text/javascript" src="jquery-1.4.3.min.js"></script>
<script type="text/javascript" src="jquery.bvalidator.js"></script>

You will aso need to include CSS file with styles for error messages:

include css:

<link href="bvalidator.css" rel="stylesheet" type="text/css" />

The web page should include elements you want to validate. This is usually a form, but validator can also work on input fields which are not inside the form. Note the data-bvalidator input field attribute which tells validator which validation actions to do for the field. The name of that attribute can be changed and this can be for example class or rel, see options for details.

HTML:

<form id="myform">
	Email: <input type="text" data-bvalidator="required,email"/>
	Url: <input type="text" data-bvalidator="url"/>
</form>

And at last you need to intialize validator for the form.

javascript:

<script type="text/javascript">
	$('#myform).bValidator();
</script>

this code should be inserted after the form or you can use jQuery's $(document).ready to execute the script as soon as the webpage's Document Object Model (DOM) has been loaded. In that case this javascript code can be before or after the form.

<script type="text/javascript">
	$(document).ready(function () {
		$('#myform).bValidator();
	});
</script>

Configuration options

Validator can be configured by passing object with options to constructor. That passed options will override default options.

<script type="text/javascript">
	
	var options = {
		singleError: true,
		showCloseIcon: false
	};

	$('#myform).bValidator(options);

</script>

OR by defining global object with options called bValidatorOptions. Validatior will check if this object exists and if does will load options from it and override default options.

<script type="text/javascript">
	
	var bValidatorOptions = {
		singleError: true,
		showCloseIcon: false
	};

	$('#myform).bValidator();

</script>

OR BOTH, you can define global bValidatorOptions object and also pass opitions to validator constructor. In that case options from bValidatorOptions object override default options and options passed as argument to constructor will override options from bValidatorOptions.

Available options:

Property Default Description
singleError false Enabling this shows only first validation error at a time.
offset {x:-20, y:-3} Fine tunes the error message tooltip position specified with the position property.
position {x:'right', y:'top'} Specifies the position of the error message tooltip. Valid values for x are: left,center,right and for y are top,center,bottom
template <div class="{errMsgClass}"><em/>{message}</div>

HTML template for the error message tooltip. This HTML code will be put inside container div element and appended after the input field. {errMsgClass} will be replaced with value of the errMsgClass propery and {message} will be replaced with error messages text.
showCloseIcon true Controls if the error message tooltip has close icon. If true user can click on it and close error message tooltip.
showErrMsgSpeed normal Error message's fade-in speed.
closeIconClass bvalidator_close_icon CSS class for error message tooltip close icon.
errMsgClass bvalidator_errmsg CSS class for error message tooltip.
errorClass bvalidator_invalid CSS class for input field with invalid value.
validClass   CSS class for input field with valid value.
lang en Language for localization.
errorMessageAttr data-bvalidator-msg Input field attribute which holds error message text. That text will override default messages or localized error messages.
validateActionsAttr data-bvalidator Input field attribute which tells validator which validation actions to do for the field.
paramsDelimiter : Delimiter for validator action options inside square brackets [].
validatorsDelimiter , Delimiter for validator actions in validateActionsAttr attribute.
validateOn null On which input field event to start validation. Values can be null, 'change', 'blur', 'keyup'
errorValidateOn 'keyup' On which input field event to start validation if error message is already shown. Values can be null, 'change', 'blur', 'keyup'

 

Callbacks

Callbacks are user defined functions that are called by validator in certain moment. Callbacks are defined the same way as configuration options. Callback functions are called for each validator action. Example of defineing callback is:

<script type="text/javascript">
	
	var options = {
		onBeforeValidate: function(element, action){
			// do something here
		},
	};

	$('#myform).bValidator(options);

</script>

Available callbacks:

Callback Arguments Description
onBeforeValidate
  1. jQuery object containing input field
  2. string validator action (example: min[5])
 Called before the validation routine starts. Returning false will cause the validation to be skipped for that validator action.
onAfterValidate
  1. jQuery object containing input field
  2. string validator action (example: max[10])
  3. validation result (true or false)
 Called after the validation routine ends. Returning false will stop executing that validator action, displaying error messages and binding events to input fields will be skipped.
onValidateFail
  1. jQuery object containing input field
  2. string validator action (example: max[10])
  3. array of all error messages for this input element
 Called after validation action fails, but before error messages are displayed to the user. Returning false will stop executing that validator action, displaying error messages and binding events to input fields will be skipped.
onValidateSuccess
  1. jQuery object containing input field
  2. string validator action (example: max[10])
 Called after validation is sucessfull. Returning false will stop executing that validator action, hiding error messages (if any) and binding events to input fields will be skipped.

 

Localization

Error messages can be localized (translated). To do that you need to pass translated messages as options to validator constructor: 

<script type="text/javascript">
	
	var options = {
		lang: 'hr',
		errorMessages: {
			hr: {
				default:    'Ispravite ovu vrijednost.',
				equalto:    'Unesite ponovno istu vrijednost.',
				differs:    'Unesite različitu vrijednost.',
				....
			}
		}
	};

	$('#myform).bValidator(options);

</script>

OR define global bValidatorOptions object with translated error messages that will be used in all validator instances:

<script type="text/javascript">
	
	var bValidatorOptions = {
		lang: 'hr',
		errorMessages: {
			hr: {
				default:    'Ispravite ovu vrijednost.',
				equalto:    'Unesite ponovno istu vrijednost.',
				differs:    'Unesite različitu vrijednost.',
				....
			}
		}
	};

	$('#myform).bValidator();

</script>

API functions

When you make instance of validator you can call some functions on that instance: 

<script type="text/javascript">
	
	var myvalidator = $('#myform).bValidator();
	
	// check if form is valid
	if(myvalidator.isValid()){
		// do somenthing
	}

</script>

or you can access validator instance through jQuery data function: 

<script type="text/javascript">
	
	$('#myform).bValidator();
	
	// check if form is valid
	if($('#myform).data('bValidator').isValid()){
		// do somenthing
	}

</script>
Function Arguments Description
getOptions   Returns the options object of the current validator instance.
isValid   Just checks if inputs in current validator instance is valid. Does not shows error messags. Calling this function is the same as calling validate(true) function.
removeErrMsg
  1. jQuery object containing inputs
 Deletes error message
getInputs   Returns all input fields associated with the validator.
bindValidateOn
  1. jQuery object containing inputs
This function is helpful when using validateOn option and dinamic forms. If you add some new input elements to form after validator is already initialized and you want to validate them on event specified with validateOn option you need to call this method. If you are not using validateOn option or you do not have dinamic forms you can forget about this function.
reset   Resets the validator. Hides all error messages like the validator is only initialized. If you have a reset button on your form this reset function is automatically bind to it.
destroy   Removes existing validator instance.
validate
  1. if true error messages are not displayed (optional)
  2. jQuery object containing inputs to validate only (optional)
 Does validation.

 

Validator actions

Validator actions are functions that checks value of input field that is validated. Here are listed built in actions but it is very simple to add custom validation functions.

Validations which validator will perform on a single input field are stated in data-bvalidator attribute by default. You can use different attribute like class or rel if you need that your HTML is valid. Just set the name of the attribute in validateActionsAttr option.

Some validator actions can have parameters and those are stated in square brackets [param1:param2:param3] after the action name.

For example:

<input type="text" data-bvalidator="maxlength[10]"/>
<input type="text" data-bvalidator="max[10],email,required"/>
<input type="text" data-bvalidator="differs[someElementID]"/>
Action Parameters Description
alpha   Checks if all characters are alphabetic.
alphanum   Checks if all characters are alphanumeric.
digit   Checks if all characters are digits.
number   Checks if value is a number.
email   Checks if value is a valid Email.
image   Checks if value ends with image file extenston like .jpg. Can be usefull for file fields.
url   Checks if value is a valid URL.
regex    
date
  1. date format
 Checks if value is a valid date in format specified by action parameter. Format can be any combination of mm,dd,yyyy with one character separator between. Example: 'mm.dd.yyyy' or 'yyyy-mm-dd'.
required  

Is a value required.

If you put required in front of other actions like data-bvalidator="required,max[10],email" then error message that field is required will be shown in error messages tooltip with other error messages.

If you put required after other actions like data-bvalidator="max[10],email,required" then error message that field is required will not be shown in error messages tooltip if there are also some more error messages.
between
  1. minimum value
  2. maximum value
 Checks is value between values in action parameters.
max
  1. maximum value

Checks is value less or equel than action parameter value.

If the input is a checkbox and it is in group of checkboxes (all with the same name for example name="chk[]") then max means "select no more then max checkboxes".

If the input is a select then max means "select no more then max options".

If you use max with checkbox or select you should use custom error messages. See errorMessageAttr option.
min
  1. minimum value

Checks is value greater or equel than action parameter value.

If the input is a checkbox and it is in group of checkboxes (all with the same name for example name="chk[]") then min means "select at least min checkboxes".

If the input is a select then min means "select at least min options".

If you use min with checkbox or select you should use custom error messages. See errorMessageAttr option.
rangelength
  1. minimum length
  2. maximum length
 Checks if length of the value is between action parameters values.
maxlength
  1. maximum length
 Checks if length of the value is smaller than action parameter value.
minlength
  1. minimum length
 Checks if length of the value is greater than action parameter value.
differs
  1. element id
 The value has to be diferent from the value of field which ID is passed as parameter.
equalto
  1. element id
 The value has to be equel to the value of field which ID is passed as parameter.
